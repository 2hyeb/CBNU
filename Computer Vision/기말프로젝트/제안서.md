# 기말 프로젝트 제안서
## 고전 컴퓨터비전 특징 기반 AI 생성 이미지 판별
### — 딥러닝 탐지기 없이, 어디까지 구분할 수 있는가

산업 컴퓨터비전 실제 · 2026 봄학기

---

## 1. 문제 정의

생성 AI(Stable Diffusion, DALL·E 등)가 만든 이미지는 사람 눈으로는 실제 사진과 구분이 거의 불가능한 수준에 이르렀다. 이로 인해 허위정보·딥페이크·학습데이터 오염 등의 문제가 커지고 있다.

본 프로젝트는 **"무거운 딥러닝 탐지기 없이, 수업에서 배운 고전 컴퓨터비전 알고리즘으로 추출한 손수 설계 특징(handcrafted feature)만으로 실제 이미지와 AI 생성 이미지를 어디까지 구분할 수 있는가"** 를 정량적으로 규명한다.

이는 단순한 분류 과제가 아니라, **"생성 모델이 이미지에 남기는 비가시적 흔적(artifact)을 어떤 고전 알고리즘이 가장 잘 포착하는가"** 를 비교·분석하는 실험적 연구다.

## 2. 왜 중요한가 (참신함·현업 관련성)

- **참신함**: 이 문제는 보통 거대 CNN/ViT로 풀며, 손수 설계 특징 기반 비교 연구는 상대적으로 드물다. "고전 방법도 의외로 통한다 / 어디서 무너지는가"를 보이는 것 자체가 결과로서 의미가 있다.
- **현업 관련성**: 제안자는 LLM/자연어처리 분야 개발자다. 생성 AI가 만든 콘텐츠를 탐지·필터링하는 문제는 멀티모달 LLM 시대의 데이터 신뢰성·안전성 과제와 직결된다. 또한 고전 특징은 가볍고 해석 가능하여, 무거운 탐지 모델을 돌리기 어려운 환경의 1차 필터로서 실용적 가치가 있다.
- **해석 가능성**: 딥러닝 탐지기는 "왜 가짜라고 판단했는지" 설명이 어렵다. 본 프로젝트의 특징(주파수 스펙트럼, 엣지 통계 등)은 의미가 명확해 결과 해석과 고찰에 유리하다.

## 3. 데이터

**CIFAKE 데이터셋** (공개, Kaggle/HuggingFace)
- 실제 이미지 60,000장 (CIFAR-10) + AI 생성 이미지 60,000장 (Stable Diffusion 1.4)
- 32×32 RGB, train/test 분할 제공 → **저해상도라 노트북에서도 빠르게 실험 가능**
- 라벨이 명확(REAL/FAKE)하여 정확도·혼동행렬 등 정량 평가가 깔끔하다.

> 데이터 미확보 상태에서도 코드가 즉시 돌도록, 주파수 아티팩트를 주입한 **합성 데이터 폴백**을 구현해 두었다(파이프라인 검증용).

## 4. 적용 알고리즘 (수업 내용 기반, 4개 이상)

모든 특징은 동일한 분류기에 입력되어 **서로 공정하게 비교**된다.

| # | 탐지 방식 (특징) | 핵심 원리 | 뿌리 강의 |
|---|---|---|---|
| 1 | **DFT 방사형 파워 스펙트럼** | 2D 푸리에 변환 후, 주파수(중심거리)별 평균 파워. 생성 모델의 업샘플링이 남기는 고주파 격자 흔적을 포착 | L4 주파수 기반 영상처리(DFT) |
| 2 | **색·히스토그램 통계** | RGB/HSV 채널별 히스토그램 + 평균/표준편차. 생성 영상의 색·채도 분포 편향 포착 | L3 히스토그램·색공간 변환 |
| 3 | **HoG (기울기 방향 히스토그램)** | 국소 gradient 방향 분포. 질감·구조의 통계적 차이 포착 | L12 객체검출기(HoG) |
| 4 | **Sobel 엣지 통계** | Sobel gradient 크기 분포·엣지 밀도. 생성 영상의 엣지 선명도/분포 차이 포착 | L4·L5 Sobel/엣지 |

**분류기 (비교 대상):**
- **SVM** (RBF 커널) — L12
- **K-NN** — L12

**비지도 베이스라인:**
- **K-means** (4개 특징 결합, 2-군집) — L6
  → "라벨 없이도 두 분포가 갈리는가?"를 확인

> 결과적으로 적용 알고리즘은 DFT·히스토그램·HoG·Sobel·SVM·K-NN·K-means로, 요구 조건(4개 이상)을 충분히 충족하며 모두 동일 문제를 위한 비교 가능한 흐름으로 구성된다.

## 5. 실험 설계

1. **특징 × 분류기 교차 비교**: 4개 특징 × 2개 분류기 = 8개 조합의 정확도/정밀도/재현율/F1/AUC 측정.
2. **혼동행렬**: 조합별 오분류 패턴 분석(실제를 가짜로 vs 가짜를 실제로).
3. **ROC 곡선**: 분류기별로 4개 특징을 한 그림에 겹쳐 판별력 비교.
4. **오분류 사례 분석**: 어떤 가짜가 안 잡히고, 어떤 실제가 오탐되는지 시각적으로 확인.
5. **(심화) 일반화 실험**: 학습에 쓰지 않은 다른 생성기/다른 데이터로 테스트하여, 손수 설계 특징의 일반화 한계를 측정 → 핵심 고찰거리.

**평가지표**: 정확도, 정밀도/재현율/F1, ROC-AUC, 혼동행렬. (이진 분류·균형 데이터라 적합)

## 6. 예상 결과와 한계 (석사 수준 고찰 포인트)

- 주파수(DFT) 특징이 GAN 대비 디퓨전에서는 흔적이 미묘해, 단독 성능은 제한적일 수 있다. 반면 HoG·Sobel 같은 질감/엣지 특징이 보완적으로 작동할 가능성.
- 손수 설계 특징의 정확도는 딥러닝(논문 보고 93~96%)보다 낮게(현실적으로 80~90%대) 나올 것으로 예상. **이는 실패가 아니라, "가볍고 해석 가능한 대신 표현력이 제한된다"는 트레이드오프를 보여주는 결과.**
- **일반화 약점**: 특정 생성기에 맞춰진 특징은 다른 생성기에 잘 안 통하는 것이 알려진 현상. 이를 실험으로 재현하면 설득력 있는 한계 분석이 된다.
- 개선 방향: 특징 결합(feature fusion), 더 큰 해상도, LBP·co-occurrence 등 추가 질감 특징.

## 7. 일정 (5일, 회사 병행 가정)

| 일차 | 작업 |
|---|---|
| 1일 | 데이터 다운로드·환경 세팅, DFT 특징 구현·확인 |
| 2~3일 | 나머지 특징(HIST/HOG/SOBEL) + SVM/KNN 학습·평가 |
| 4일 | 전체 실험 실행, 표·혼동행렬·ROC 정리, 오분류 분석 |
| 5일 | (심화) 일반화 실험 + 발표자료·보고서 문장 정리 |

> 무거운 연산은 라이브러리(OpenCV/scikit-learn/scikit-image)가 처리하므로, 코딩량보다 **특징 설계 근거와 결과 해석**이 핵심 작업이다. 스타터 코드가 이미 동작 검증되어 있어 1~3일차 부담이 크게 줄어든다.

## 8. 발표·보고서 흐름

문제(생성 AI 탐지) → 왜 중요한가 → 데이터(CIFAKE) → 어떤 고전 알고리즘을 왜 적용했나 → 결과(표·ROC·혼동행렬) → 결과 해석(어떤 특징이 왜 잘/못 했나) → 한계(일반화) → 개선 방향.

## 9. 산출물

- `code/` : 동작 검증 완료된 스타터 코드 (dataset.py / features.py / run_experiment.py)
- `results/` : 결과표(CSV), 혼동행렬, ROC, 오분류 사례 (현재는 합성 데이터 기준 예시)
- 실제 CIFAKE를 받아 `python run_experiment.py --data <경로>` 로 교체 실행하면 본 실험 결과 생성.
